# Ruimtemeesters Chatbot — Brand Pass 2

**Date:** 2026-04-17
**Status:** Spec
**Author:** Ralph (via proactive-brainstorm)
**Scope:** Finish the on-brand customization of the chatbot after Phase A1+A2 (commit `c38504d74`) landed the first rebrand.

## Why

Phase A1+A2 nailed the core palette, splash, prompt suggestions, and theme CSS. Baseline measurements from 2026-04-17 show 8 gaps remaining — mostly surfaces that bypass the theme CSS (PWA manifest, browser tab, OS-level share targets, assistant avatars, personalized greeting). These are the moments where the bot still "feels" like stock OpenWebUI despite the rebrand. Closing them makes the product feel coherent across tab, splash, chat, settings, and mobile home-screen installs.

## Non-goals

- Refactoring OpenWebUI's theming primitives.
- Localisation beyond what already exists (NL-first with en-US fallback).
- New assistants — existing 5 are in scope for avatars only.
- Changing LLM providers beyond setting a default model (Gemini wiring landed in a prior session).

## In-scope changes

### 1. Fix broken `WEBUI_FAVICON_URL`

**Problem:** `docker-compose.rm.yaml` sets `WEBUI_FAVICON_URL=/brand-assets/favicon.svg`, which doesn't exist on disk. SvelteKit's SPA fallback returns `index.html` with `Content-Type: text/html`, so the browser silently fails to use it. The env var is also read by `backend/open_webui/utils/webhook.py` for activity-image notifications — broken there too.

**Change:** Point to `/brand-assets/icon-blue.png` (existing real file).

### 2. Custom PWA manifest

**Problem:** `GET /manifest.json` is generated at runtime by `backend/open_webui/main.py:get_manifest_json` with hardcoded `background_color=#343541`, `icons=[/static/logo.png]`, name suffixed `" (Open WebUI)"`, English description. This surfaces on Android "Add to Home Screen", iOS splash, and browser install prompts.

**Change:** Add `static/brand-assets/rm-manifest.json` with RM brand values, then set `EXTERNAL_PWA_MANIFEST_URL=http://localhost:8080/static/brand-assets/rm-manifest.json` in compose so the backend proxies it. No Python patch — upstream-clean.

### 3. Persistent trust banner

**Problem:** No signal that this is a private RM workspace with data guarantees.

**Change:** Add `WEBUI_BANNERS` JSON in compose with a single non-dismissible info banner:

```
Besloten werkomgeving voor Ruimtemeesters · Data wordt niet gedeeld met LLM-providers
```

### 4. Default model = Gemini 2.5 Flash Lite

**Problem:** No `DEFAULT_MODELS` set; users land on the first-alphabetical model in the picker.

**Change:** `DEFAULT_MODELS=gemini.gemini-2.5-flash-lite` in compose. Verified live against Google's endpoint in the Gemini-wiring session.

### 5. Per-assistant avatars

**Problem:** All 5 RM assistants (Beleidsadviseur, Demografie Analist, Ruimtelijk Adviseur, Sales Adviseur, Assistent) share `/brand-assets/icon-blue.png`. Confirmed via `SELECT DISTINCT meta::jsonb->'profile_image_url'` returning 1.

**Change:** Author 5 SVGs at `static/brand-assets/assistants/{policy,chart,map-pin,currency,spark}.svg` — each uses `--rm-primary` (Klein Blue) on Smart White circle. Update `rm-tools/register_assistants.py` to reference them. Re-run registration script. (Spec originally called for a handshake icon for Sales Adviseur; Heroicons doesn't ship one, so the `currency` icon — Heroicons currency-dollar — was used instead: unambiguously represents sales/revenue.)

### 6. Disable community sharing

**Problem:** Internal tool with gemeente/client data; community-share button is a data-leak path.

**Change:** `ENABLE_COMMUNITY_SHARING=false` in compose.

### 7. Personalized landing greeting

**Problem:** Stock `Placeholder.svelte` shows model name(s) with no context. Ralph's name is available via the `user` store (populated from Clerk JWT session).

**Change:** In `src/lib/components/chat/Placeholder.svelte`, above the prompt-suggestions block and only when `history.messages.length === 0` and `!$selectedFolder` and `$user?.name`, render:

```
Hoi {firstName}, waarmee kan ik je vandaag helpen?
```

where `firstName = $user.name.split(' ')[0]`. Fall back to nothing if no name.

### 8. About modal copy

**Problem:** `SettingsModal.svelte:848` shows stock "About".

**Change:** Keep the i18n key but patch the translation / content block below it to add one line: _"Ruimtemeesters AI · Gebouwd op Open WebUI · Ondersteund door Gemini, OpenAI, Claude en lokale Ollama modellen."_

### 9. Seed-script for Gemini connection

**Problem:** Gemini model curation (5 models, `gemini.` prefix, "Google" tag) lives in `config` table from a manual `POST /openai/config/update`. A DB wipe loses it.

**Change:** `scripts/seed-gemini-connection.sh` that mints an admin JWT and POSTs the curation. Runnable on first boot or after `docker compose down -v`. Document in `.env.rm.example`.

## Validation plan

3. Login as admin via JWT cookie shortcut → visit `/` → assert greeting text

## Risks

| Risk                                                                                   | Likelihood | Mitigation                                                                                                                                                                                |
| -------------------------------------------------------------------------------------- | ---------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `EXTERNAL_PWA_MANIFEST_URL` with localhost:8080 breaks when accessed via reverse proxy | medium     | Use relative URL fetched server-side; test both direct and via nginx. If it breaks, fall back to patching `main.py:get_manifest_json` (upstream-drift accepted for this single file).     |
| Personalized greeting leaks user's name in screenshot/screen-share contexts            | low        | First name only, no email. Matches what assistants already show in chat.                                                                                                                  |
| Svelte patch in `Placeholder.svelte` conflicts with future upstream merge              | medium     | Keep edit minimal (5–10 lines), use a feature-flag prop that defaults to off so upstream behavior is unchanged when Svelte diff reverts. Comment with file:line pointer for quick rebase. |
| 5 distinct SVG avatars look inconsistent visually                                      | low        | Use single set from Lucide / Heroicons (both MIT), stroke-only, brand color fill. Consistency guaranteed by source family.                                                                |
